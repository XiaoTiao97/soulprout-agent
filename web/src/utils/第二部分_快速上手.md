## 2.1 第一个对话（通用模式）

### 什么是通用模式？

通用模式就是萌芽最基础的对话方式——你提问，萌芽回答。

但和普通的AI聊天不同，萌芽会**自动判断**是否需要调用工具来帮助你。比如：
- 你问"明天北京天气怎么样？" → 萌芽自动调用天气查询工具
- 你说"帮我查一下最近关于AI的新闻" → 萌芽自动联网搜索
- 你问"从公司到机场怎么走？" → 萌芽自动打开地图规划路线

**你不需要告诉它"去查资料"，它会自己判断需要什么工具。**

### 实战：完成你的第一个对话

让我们从一个简单的例子开始。

#### 步骤1：进入对话界面

打开浏览器，访问 **www.mengya.chat**，登录后，你会看到这样的界面：

![主界面](/docs-images/main-interface.png)

*说明：中间是对话区域，底部是输入框，左侧是四大功能库的入口*

#### 步骤2：输入你的第一个问题

在底部的输入框里，输入：

```
我想了解一下特斯拉公司最近的股价走势，以及市场上有哪些相关的新闻和分析。
```

*说明：在底部输入框输入问题，按回车或点击发送按钮*

#### 步骤3：观察萌芽的回应

发送后，你会看到萌芽开始工作：

![思考过程](/docs-images/thinking-process.png)

*说明：萌芽会显示"正在思考..."，然后列出它的规划（PLAN），比如：*
- *步骤1：查询特斯拉最新股价*
- *步骤2：搜索近期相关新闻*
- *步骤3：整合信息并给出分析*

**注意这个细节**：萌芽没有直接给答案，而是先给你看它的"思考过程"——这就是萌芽的**交互式规划**能力。它会告诉你："我打算这样帮你解决问题，你觉得可以吗？"

#### 步骤4：查看结果

确认规划没问题后（通常直接等它执行即可），萌芽会：

1. **自动调用金融工具**，查询特斯拉股价
2. **自动调用搜索工具**，查找相关新闻
3. **整合所有信息**，给你一个完整的分析报告

这就是通用模式的核心：**像和一个聪明的助理聊天一样自然，但它能自动调用各种工具帮你完成任务。**



## 2.2 界面导览

完成了第一次对话，现在让我们熟悉一下萌芽的界面布局。

### 整体布局

萌芽的界面分为四个主要区域：


#### 左上角：四大功能库

这是萌芽的核心功能区入口：

![四大功能库](/docs-images/four-libraries.png)


| 功能库 | 作用 | 什么时候用 |
|---|---|---|
| **工具库 (MCPs)** | 查看和管理各种工具（搜索、地图、金融、法律等） | 想了解萌芽有哪些工具，或给特定智能体绑定工具 |
| **知识库 (Knowledges)** | 上传和管理长期复用的文档（公司资料、行业报告等） | 需要让AI参考特定资料时 |
| **专家库 (Agents)** | 查看和管理智能体（专家团队） | 想使用或编辑AI专家团队 |
| **技能库 (Skills)** | 管理AI技能包（开发中） | 功能上线后使用 |

**新手提示**：刚开始用的时候，你可以先不急着点这些，直接在中间对话区聊天就行。等熟悉了基础操作，再慢慢探索这些高级功能。

#### 中间区域：对话区

这是你最主要的工作区域，显示你和萌芽的聊天记录：

- 你的提问和萌芽的回答
- 萌芽的思考过程（PLAN）
- 工具调用的结果

#### 底部：输入区域（重点）

这是萌芽最强大的功能区，让你可以精细控制每一次对话：

![输入区域](/docs-images/input-area.png)
![输入区域](/docs-images/agent-area.png)

*说明：标注模型选择、工具、上传文件、知识库、专家库选择器的位置*

**他们分别是：**

| 功能 | 说明 |
|---|---|
| **上传文件按钮**（📎） | 上传本地文件给萌芽看。支持PDF、Word、Excel、PPT、txt、md等多种格式，上传后可以直接基于文件内容提问 |
| **模型选择器**（如：kimi-k2.5-thinking） | 选择使用哪个AI模型来回答你的问题。不同模型有不同的特点：有的擅长推理，有的响应速度快。默认会根据任务自动选择，但你也可以手动指定 |
| **工具按钮**（🔧 工具） | 查看当前可用的工具，可以为本次对话绑定特定工具。通常不需要手动选择，萌芽会自动匹配 |
| **知识库选择器**（📚 知识库） | 为本次对话绑定已上传的知识库，萌芽会参考知识库中的资料来回答。适合需要引用特定领域资料的场景 |
| **专家库选择器**（👥 专家库） | **这是萌芽的特色功能！**选择已创建的专家或专家团队进行对话，选中的专家会使用预设的配置（模型、工具、知识库、系统提示词），相当于直接召唤一位"专业顾问"为你服务 |

**💡 使用技巧**：这些选项不是必须设置的。默认情况下，你直接在输入框打字发送就行。当你需要更精细的控制时，再使用这些选项。

#### 右侧：工具信息区 / 文件库（可切换）

右侧边栏可以通过顶部标签切换显示两种内容：

![右侧边栏](/docs-images/right-sidebar.png)

*说明：顶部有两个标签页"工具信息"和"文件库"，点击切换显示内容*

| 标签页 | 内容 | 价值 |
|---|---|---|
| **工具信息** | **Plan（规划）**：显示萌芽对任务的整体规划，包括要执行的步骤和思路<br><br>**Tools（工具）**：显示调用了哪些工具、调用的参数和返回结果<br><br>**SubAgents（子智能体）**：显示创建和调用的子智能体及其执行状态 | 让你知道AI正在做什么、怎么做的；查看工具调用的具体参数和结果；追踪每个子智能体的工作进度；如果发现问题，可以及时干预和调整 |
| **文件库** | 显示当前对话中已上传的所有文件：<br>• 点击文件名可以查看文件内容<br>• **支持任意常见格式的显示**（PDF、Word、Excel、PPT、图片、代码文件等）<br>• 对于txt、md、py等文本格式，支持在线编辑<br>• 萌芽可以直接读取文件内容进行分析和处理 | **人机协同编辑**：在文件库中打开文件，你可以直接修改内容，然后告诉萌芽"按我修改的版本继续"，无需复制粘贴 |

### 界面使用小贴士

**💡 右侧边栏可以收起**：如果屏幕较小或不需要查看执行过程，可以点击右侧边栏的边缘收起它，让对话区更宽敞。

**💡 关注Plan区域**：当萌芽处理复杂任务时，先看看它的Plan是否合理。如果方向不对，及时纠正可以节省大量时间。

**💡 文件库即传即用**：上传文件后，直接在输入框说"请分析这份报告"即可，萌芽会自动读取文件内容。
