# 第五部分：Soul 模式

Soul 模式是 Soulprout 的完整体，集长期记忆、上下文压缩、多端触达与深度办公协作于一体，让 AI 真正成为你身边持续陪伴的伙伴。

---

## 5.1 Soul 模式的核心能力

### 持续记忆

Soul 模式会主动积累对你的理解，不因对话结束而遗忘：

- **用户信息记忆**：你的职业、兴趣、工作风格、偏好习惯
- **智能体个性化**：你可以为它定义名字、性格和说话风格
- **业务上下文记忆**：通过记忆工具将重要信息持久化，下次对话自动召回

每一次交流都在加深它对你的理解，越用越懂你。

### 上下文压缩

长期对话不会让 Soulprout 退化。Soul 模式内置上下文压缩机制：

- 对话超出窗口阈值时自动触发压缩
- 保留关键信息，清除冗余内容
- 你也可以随时手动触发（通过 `conversation_option` 工具）
- 始终保持清晰、高效的对话上下文

### 多端触达

Soul 模式支持通过多种渠道与你的 Soulprout 沟通：

| 渠道 | 说明 |
|---|---|
| **网页端** | 浏览器直接访问，随时开始对话 |
| **桌面客户端** | 下载安装专属桌面应用，像使用本地软件一样流畅（详见 Gateway 部分） |
| **微信** | 通过微信接入，随手发消息即可与 Soulprout 对话 |
| **飞书** | 企业飞书机器人，深度融入工作协作流 |
| **企业微信** | 企业微信群机器人，无缝接入内部沟通场景 |
| **小爱音箱** | 语音交互，解放双手，随口问随口答 |
| **Rokid AI 眼镜** | 可穿戴端接入，真正的随身 AI |

无论你习惯用哪个平台，Soulprout 都能出现在那里。

## 5.2 人机协同编辑

### 告别复制粘贴

传统 AI 工作流的痛点：在 AI 网站生成内容 → 复制到本地文档修改 → 再粘贴回 AI → 循环往复。

**Soulprout 直接消灭这个循环。**

### 使用方式

**第一步**：在对话底部上传文件（支持 PDF、Word、Excel、PPT、txt、md、py 等）

**第二步**：让 Soulprout 基于文件内容生成或修改内容

**第三步**：在右侧"文件库"中直接打开文件，修改文字、调整结构、增删内容

**第四步**：回到对话框说"我改了一下，请按我修改的版本继续"——Soulprout 自动读取你的修改继续工作

**全程无需切换页面，无需复制粘贴。**

### 适用场景

- **写报告**：AI 生成初稿 → 你润色修改 → AI 继续补充完善
- **改代码**：AI 写代码 → 你调试修改 → AI 继续优化
- **整理资料**：AI 整理结构 → 你调整细节 → AI 继续完善

## 5.3 蓝图模式（行动规划）

### 从根源避免"答非所问"

当任务复杂、信息不全或目标模糊时，普通 AI 会急于生成内容——方向一开始就错，后面的工作全白费。

**蓝图模式让 Soulprout 先规划、再执行。**

### 工作方式

遇到复杂任务时，Soulprout 会先：

1. **梳理信息缺口**："我还需要确认……"
2. **澄清模糊意图**："你说的 XX 是指 A 还是 B？"
3. **生成结构化蓝图**：呈现完整的执行方案——分析任务背景、明确真实意图、列出执行步骤、标注每步使用哪些工具/技能
4. **确认后执行**：你认可规划后，按蓝图高效执行

### 蓝图的价值

| 对比 | 传统方式 | 蓝图模式 |
|---|---|---|
| **执行质量** | 基于 AI 的猜测执行，可能跑偏 | 基于完全对齐的需求执行 |
| **修改成本** | 结果不满意反复重来 | 规划阶段纠偏，一次到位 |
| **透明度** | 黑盒执行 | 方案可见，可在执行前干预 |

### 实际体验

当你说：
```
帮我规划一次团队建设活动。
```

Soulprout 不会立刻给方案，而是先生成蓝图：
- 确认人数、预算、时间
- 明确偏好（户外 / 室内？餐饮偏好？）
- 列出规划步骤和使用的工具

确认蓝图没问题后，再执行并输出完整方案。

## 5.4 三大功能协同

Soul 模式、人机协同编辑和蓝图模式不是孤立的，它们共同构成 Soulprout 的深度办公体验：

```
蓝图模式            →  先规划，确保方向正确
    ↓
人机协同编辑         →  执行中你可以随时介入修改
    ↓
Soul 模式记忆        →  每次协作都在积累你的偏好和习惯
    ↓
下次对话更懂你，更准确
```
