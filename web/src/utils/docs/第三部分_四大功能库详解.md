# 第三部分：四大功能库详解

四大功能库是 Soulprout 区别于普通 AI 工具的核心能力基础。掌握它们，才能真正发挥 Soulprout 的生产力潜力。

---

## 3.1 专家库（Agents）

专家库是 Soulprout 最核心的功能之一，用于**创建、管理和复用 AI 智能体**。

### 智能体的构成要素

每个智能体都可以独立配置：

| 参数 | 说明 |
|---|---|
| **类型** | 主智能体（统筹调度）/ 子智能体（专注子任务） |
| **模型** | 为任务特性选择最合适的 AI 模型 |
| **系统提示词** | 定义角色、能力、工作流程和输出规范 |
| **工具** | 绑定该智能体可以使用的工具 |
| **知识库** | 绑定专属的参考资料库 |
| **技能** | 加载特定的技能包，赋予专业能力 |
| **子智能体** | 主智能体可配置要调度的下属子智能体 |

### 主智能体 vs 子智能体

**子智能体**：专注于某个具体环节的"专员"，例如"数据搜集员"、"文案撰写员"、"代码审查员"。

**主智能体**：负责统筹调度的"项目经理"，接收用户任务、分解给子智能体、整合输出最终结果。

### 如何创建智能体？

**方式一：通过专家模式自动生成（推荐）**
1. 在对话中描述业务需求
2. Soulprout 交互确认细节后自动规划团队
3. 确认后所有智能体自动保存到专家库

**方式二：手动创建**
1. 点击"+"新建，填写名称、描述、系统提示词
2. 选择模型，配置工具、知识库、技能
3. 保存即可随时调用

## 3.2 知识库（Knowledges）

知识库是 Soulprout 的**资料管理中心**，适合管理海量文档并长期复用。

### 知识库 vs 直接上传文件

| 场景 | 推荐方式 |
|---|---|
| 资料多且杂，只需检索相关片段 | 知识库 |
| 资料少（1-3份），需要 AI 完整理解全文 | 直接上传文件 |

### 双模式检索

| 模式 | 适用场景 |
|---|---|
| **RAG 快速检索** | 简单信息查询、快速问答，响应速度快 |
| **AI 知识库检索智能体** | 复杂需求、综合分析，通过段落摘要索引深度挖掘资料 |

### 典型使用场景

- 客服智能体 → 绑定"产品知识库"
- 法律顾问智能体 → 绑定"法律法规库"
- 行业分析师智能体 → 绑定"行业报告库"

## 3.3 工具库（Tools）

工具库是 Soulprout 的**能力扩展中心**，赋予智能体"动手做事"的能力。

### 内置工具能力全览

Soulprout 智能体可拥有以下能力：

**文件操作能力**
- 读取工作区文件（docx / pdf / pptx / xlsx / txt / md / py / json / html 等）
- 写入和追加文本文件
- 文本内容精确替换编辑
- 图片内容识别与查看

**代码与命令执行**
- 在工作区执行 Shell 命令（Bash / Python / Node.js 脚本等）
- SaaS 模式下运行于隔离沙箱，本地部署模式直接在服务器执行

**互联网能力**
- 联网搜索（web_search）
- 抓取 URL 并转换为 Markdown（web_fetch）

**智能体协作**
- 创建新智能体并永久保存到专家库（create_agent）
- 临时召唤子智能体立即执行任务（call_sub_agent）
- 行动蓝图规划——调用规划专家对复杂任务进行结构化分解（get_action_blueprint）
- 查询可用模型、工具、专家列表（list_info）

**记忆管理**
- 创建、检索、加载、编辑、删除记忆条目（base_memory / create_memory / edit_memory）
- 用户个性化配置持久化，包括用户信息和智能体形象偏好（user_option）

**知识库操作**
- 统一知识库工具（knowledge_base）：RAG 检索（module=search）、Chunk 摘要（module=chunk_abstract）、段落全文（module=chunk_content）、知识库列表（module=kb_info）

**技能管理**
- 搜索、查看、加载、关闭技能（skills）

**上下文管理**
- 清除或压缩当前对话上下文（conversation_option）

**交互反馈**
- 向用户发起批量提问，收集选择或文本输入（ask_user_feedback）

### 智能工具匹配

你**无需手动选择工具**，Soulprout 会自动理解任务需求并调用最合适的工具组合。当然，你也可以在创建智能体时为其精确绑定特定工具，限定其能力边界。

## 3.4 技能库（Skills）

技能库是 Soulprout 的**专业能力组件库**，已正式上线。

### 什么是技能（Skill）？

Skill 是一种可复用的专业能力包，包含完成某类任务所需的脚本、工作流、最佳实践和工具调用规范。加载一个 Skill，智能体立即具备该领域的专业执行能力，无需每次从零描述流程。

### 当前已有技能

| 技能名 | 能力描述 |
|---|---|
| **docx** | Word 文档的专业生成与编辑，包括格式控制、修订追踪 |
| **xlsx** | Excel 表格的创建、数据处理与公式计算 |
| **pptx** | PowerPoint 演示文稿的生成与编辑 |
| **pdf** | PDF 文件的内容提取、表单填写与处理 |
| **canvas-design** | 富文本画布设计，支持自定义字体与视觉排版 |
| **frontend-design** | 前端页面设计与 HTML/CSS 组件开发 |
| **web-artifacts-builder** | Web 应用脚手架与交互式网页构建 |
| **webapp-testing** | Web 应用自动化测试 |
| **doc-coauthoring** | 文档多人协作编辑流程支持 |
| **internal-comms** | 企业内部沟通内容生成（FAQ、公告、通知等） |
| **brand-guidelines** | 品牌规范文档的生成与维护 |
| **theme-factory** | UI 主题样式生成与管理 |
| **mcp-builder** | MCP 工具开发与评估 |
| **claude-api** | 调用 Claude API 进行 AI 开发（含 Python/Ruby 示例） |
| **slack-gif-creator** | Slack 平台 GIF 内容创作 |
| **algorithmic-art** | 算法艺术图像生成 |
| **skill-creator** | 创建新技能包，扩展技能库 |

### Skill 的来源

- **系统技能**：平台内置，通过向量检索召回最相关技能
- **个人技能**：用户自定义创建，直接全量可见
- 两者均可绑定到智能体，个人技能优先级高于系统技能

## 3.5 四大库如何协同工作？

### 案例：生成行业研究报告

| 功能库 | 协作内容 |
|---|---|
| **专家库** | 主智能体"报告调度中心"统筹，调度"数据搜集员"、"政策分析员"、"竞品分析师"、"报告撰写员"四个子智能体并行工作 |
| **工具库** | 数据搜集员调用 web_search / web_fetch 获取最新信息；报告撰写员调用 bash 执行脚本生成文档 |
| **知识库** | 所有智能体共享"行业报告库"，使用 AI 知识库检索智能体深度挖掘历史资料 |
| **技能库** | 报告撰写员加载 docx 技能，按标准结构生成规范的 Word 报告 |

### 协同工作流

```
用户任务
    ↓
专家库（主智能体）—— 蓝图规划与任务分解
    ↓
子智能体1 ←→ 工具库（web_search）
子智能体2 ←→ 知识库（行业资料）
子智能体3 ←→ 技能库（docx 写作）
    ↓
主智能体 —— 整合输出
    ↓
用户获得完整结果
```
