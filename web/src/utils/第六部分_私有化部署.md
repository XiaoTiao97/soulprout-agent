# 第六部分：私有化部署

> 把萌芽部署在企业的服务器上，让它成为企业真正的Agent生产力。

***

## 6.1 为什么选择私有化部署？

**核心原因：企业内部数据有安全要求，通用Agent平台进不了企业的内部系统，唯有把Agent直接部署在企业服务器或内网，才能解决这个问题。**

现在市面上的Agent平台，员工用来查公开资料、写个文档、问个常识问题都没问题。但如果你想让它查公司内部的客户数据、跑个内部业务脚本、读个保密的内部文档？根本做不到——因为它们跑在公网上，连不进你们企业的内网和业务系统。

当萌芽部署在企业自己的服务器上后，它能实现：

| 核心能力 | 具体说明 |
| ---- | ---- |
| **访问内部数据** | 直接连接企业数据库、API、业务系统，Agent可以查询真实的业务数据 |
| **定制品牌形象** | 替换为企业专属logo和VI视觉，员工打开的就是"咱们公司自己的Agent平台" |
| **满足合规要求** | 所有数据都留在内网，不流出企业，满足金融、医疗、政务等行业的安全合规要求 |

**一句话总结：私有化部署 = 萌芽的成熟能力 + 你们企业的品牌 + 你们企业的内部数据**

***

## 6.2 私有化部署的三大能力

### 6.2.1 页面定制

**为什么要做品牌定制？**
打开一个自己企业品牌标识的专属Agent平台，归属感完全不一样。如果用来对外展示，也是企业数字化能力的直接体现。

**可定制内容：**

* 登录页、导航栏、内部Agent名称替换为企业logo
* 主题色、界面风格按照企业VI规范定制
* 功能模块可以按需调整，匹配企业实际使用场景

**带来的价值：**

* **员工视角**：用的是"公司自己的平台"，不是外来工具，归属感更强
* **企业视角**：相当于拥有了7×24小时不休息的Agent员工队伍，处理标准化任务
* **品牌视角**：客户来访、对外展示时，是企业数字化能力的直观体现

不是简单给员工配了个助手，而是在搭建企业自己的Agent生产力体系。

***

### 6.2.2 工具定制

**能不能调用内部数据，是通用Agent和私有化部署Agent的本质区别——两者的能力差了好几个量级。**
公网上的通用Agent只能访问互联网公开信息，对企业内部的数据和业务一窍不通。私有化部署的萌芽可以通过MCP工具配置，和企业的内部系统完全打通：

* 对接内部数据库：Agent可以直接查询核心业务数据，自动生成精准的报表和分析
* 对接企业API：Agent可以直接调用内部服务，执行实际的业务流程
* 对接业务脚本：Agent可以运行企业自定义的业务逻辑

**举个完整的场景对比案例：**

| 需求："帮我出一份Q2华南区的客户续约分析报告，按公司模板整理好，走审批流程提交给部门总监" |
| ---------------------------------------------- |
| **通用Agent的回答：**<br>"抱歉，我无法访问你们的内部销售数据，也不知道你们的报告模板格式是什么，更没法帮你走内部审批流程。我可以给你一份通用的报告模板参考。" |
| **私有化部署的能力：**<br>1\. 直接读取内部销售数据库，拉取Q2华南区所有客户的续约数据<br>2\. 调取内部文档库的报告模板，按照要求整理成规范的分析报告<br>3\. 调用内部OA系统的审批API，自动提交给部门总监<br>4\. 告诉你："报告已经生成并提交审批，总监审批后会通知你。" |

**简单说：通用Agent是只能给建议的"外部顾问"，私有化部署的萌芽是能直接干活的"内部员工"。**

***

### 6.2.3 技能定制

现在通用领域已经有上万个现成的Skill可以直接用，但每个企业内部都有自己独特的工作流程、行业术语、业务规范，这些个性化的东西，通用Skill肯定覆盖不到。

私有化部署支持企业根据业务需求自定义Skill，扩展Agent的专业能力：

* 把行业的专业术语、业务规范封装成Skill，让Agent说"行业内的话"，不会出专业错误
* 把企业内部的标准操作流程（SOP）做成Skill，让Agent自动按照你的规则处理任务
* 随着业务发展，可以持续更新和新增Skill，Agent会越来越好用

**最终效果：**
相当于把企业的行业专家、老员工的经验都装进了Agent里，新人哪怕刚入职，用这个Agent也能做出符合专家水平的工作成果。

## 6.3 两种部署方案，按需选择

私有化部署的页面定制、工具开发、Skill开发，需要前端开发、后端开发、AI应用工程师等不同技术岗位的人员配合完成。

不是所有企业都有这样的技术团队，单独招这些人成本又很高，所以我们提供两种方案：

| 方案一：企业自主修改 | 方案二：萌芽一站式服务 |
| ---------- | ----------- |
| 如果有自己的技术团队，可以拿到部署包后，自己修改前端、开发工具、定制Skill，我们提供技术支持 | 如果没有技术团队，所有的定制开发、部署上线工作全部交给我们来做，企业只需要提需求就行 |

### 选择一站式服务，可以这样做：

* 如需页面定制：提供企业logo、样式设计
* 如需工具开发：提供需对接系统的接口说明
* 如需技能定制：描述清楚业务场景和需求

### 我们负责交付：

| 模块 | 交付内容 |
| --- | ---- |
| **前端定制** | logo替换、主题色配置、界面风格调整 |
| **工具对接** | 开发MCP工具，完成与内部数据库/API/脚本的对接 |
| **Skill开发** | 根据业务需求开发定制化技能包 |
| **部署上线** | 在企业的Linux服务器上完成部署、测试、上线 |
| **培训支持** | 使用培训、技术支持、后续迭代升级 |

**服务流程：** 需求沟通 → 方案报价 → 开发交付 → 上线培训 → 持续支持

## 6.4 适合私有化部署的企业

* ✅ **数据敏感行业**：金融、医疗、政务、制造等，对数据安全要求高
* ✅ **内部系统复杂**：有大量数据库、API、业务系统需要Agent整合
* ✅ **注重品牌形象**：希望统一展示企业数字化能力
* ✅ **业务需求独特**：通用Agent无法满足行业特定需求
* ✅ **规模化使用**：全公司或部门级推广，需要统一管控
