# 第四部分：专家模式实战

你好！欢迎来到第四部分。

在前面的学习中，你已经掌握了萌芽的基础操作和四大功能库。现在，让我们进入萌芽最强大的功能——**专家模式**。

这是萌芽区别于普通AI工具的核心能力，也是真正能帮你提升工作效率的利器。

---

## 4.1 什么是专家模式？

### 从一个比喻开始

想象你要装修房子，你有几种选择：

| 选择 | 方式 | 结果 |
|---|---|---|
| **选择一：自己动手** | 一个人干所有活：设计、水电、木工、油漆…… | 累得半死，效果还不一定好 |
| **选择二：请一个全能工人** | 一个人包办所有工种 | 什么都会一点，但都不精通 |
| **选择三：组建专业团队** | 设计师（整体规划）+ 水电工（隐蔽工程）+ 木工（定制家具）+ 油漆工（墙面装饰）+ 项目经理（统筹协调） | 专业的人做专业的事，又快又好 |

**专家模式，就是第三种选择。**

### 专家模式的核心思想

面对复杂任务，萌芽不会让一个AI"单打独斗"，而是：

1. **分析任务复杂度**：理解需要哪些能力
2. **组建专家团队**：为每个子任务匹配专门的AI智能体
3. **并行协作处理**：多个专家同时工作，效率翻倍
4. **统一调度整合**：由主智能体统筹协调，输出完整结果

### 专家模式 vs 通用模式

| 对比项 | 通用模式 | 专家模式 |
|---|---|---|
| **适合任务** | 简单问答、单步骤任务 | 复杂项目、多步骤任务 |
| **工作方式** | 单个AI处理 | 多个AI协作 |
| **执行效率** | 串行处理 | 并行处理 |
| **结果质量** | 依赖单一AI能力 | 各取所长，综合优化 |
| **透明度** | 黑盒，看不到过程 | 白盒，全程可见每个步骤 |

**简单说**：
- 通用模式 = 问AI一个问题，它直接给你答案
- 专家模式 = 你提出需求，萌芽组建团队，分工协作完成项目

## 4.2 5分钟组建专家团队

这是最激动人心的部分！你只需要描述需求，萌芽会自动帮你组建专家团队。

### 实战演示：组建"基金筛选助手"团队

让我们以"短期基金筛选"为例，看看专家模式的完整流程。

#### 第一步：点击定制专家团队

在对话页面点击“定制专家团队”

![输入需求](/docs-images/create-agent.png)

你可以看到AI在和你解释如何定制专家。

接下来在对话框中输入：

```
我要做一个短期基金的筛选助手，需要帮我筛选出近期表现好的基金。
```

#### 第二步：AI交互式确认需求

萌芽不会立刻开始干活，而是先和你确认细节：

![交互确认](/docs-images/interactive-confirm.png)

*说明：萌芽列出需要确认的问题，如：*
- *筛选周期是多长？（1周/1个月/3个月？）*
- *关注哪些行业？（科技/医疗/消费/全部？）*
- *筛选指标有哪些？（收益率/风险等级/规模？）*
- *输出格式是什么？（表格/报告/列表？）*

你需要回答这些问题，确保萌芽完全理解你的需求，从根源避免"答非所问"。

#### 第三步：AI自动规划专家团队

确认需求后，萌芽会展示它规划的团队结构：

![团队规划](/docs-images/team-planning.png)

*说明：萌芽列出规划的专家团队，包括：*

| 角色 | 名称 | 职责 | 配置 | 输出 |
|---|---|---|---|---|
| **🎯 主智能体** | 基金筛选调度中心 | 协调整个筛选流程，整合各专员输出 | 模型：kimi-k2.5-thinking（擅长推理规划） | 最终筛选报告 |
| **👤 子智能体1** | 市场热点侦查员 | 搜索当前政策利好、行业风口、市场热点事件 | 工具：互联网搜索工具 | 热点方向列表 |
| **👤 子智能体2** | 基金数据猎手 | 获取基金基础信息、近期表现数据 | 工具：金融数据查询工具 | 基金基础数据表 |
| **👤 子智能体3** | 短期动量分析师 | 分析基金近期技术走势、涨跌动量 | 工具：数据分析工具 | 技术分析报告 |
| **👤 子智能体4** | 综合评估员 | 结合热点、数据、分析，给出最终推荐 | 模型：Deepseek（擅长综合分析） | 筛选结果和投资建议 |

#### 第四步：确认并生成团队

查看团队构成，如果满意，点击"确认生成"。

萌芽会自动：
- ✅ 为每个智能体配置最优模型
- ✅ 绑定所需的工具库
- ✅ 生成专业的系统提示词
- ✅ 设置子智能体之间的调用关系
- ✅ 将所有智能体保存到专家库

整个过程你只需要点几下鼠标，**不需要写一行代码，不需要懂AI技术**。

#### 第五步：立即使用

团队创建完成后，你可以在输入区域的"专家库"选择器中找到"基金筛选调度中心"。

选择它，然后输入：

```
帮我筛选一下最近适合投资的基金。
```

主智能体会自动：
1. 调度"市场热点侦查员"搜索热点
2. 调度"基金数据猎手"获取数据
3. 调度"短期动量分析师"进行技术分析
4. 收集所有子智能体的输出
5. 整合生成最终的筛选报告

![专家模式运行](/docs-images/expert-mode-running.png)

*说明：展示主智能体调度子智能体，并行工作的过程*

**⏱️ 整个过程只需要5分钟！**

## 4.3 专家团队的运作原理

了解了如何使用，让我们深入看看专家模式背后的运作机制。

### 架构图解

```
用户
  ↓ 提出需求
主智能体（调度中心）
  ↓ 任务分解
┌─────────┬─────────┬─────────┐
│ 子智能体1 │ 子智能体2 │ 子智能体3 │
│ 市场侦查  │ 数据获取  │ 分析评估  │
│ 并行执行  │ 并行执行  │ 并行执行  │
└─────────┴─────────┴─────────┘
  ↓ 结果汇总
主智能体（整合输出）
  ↓ 最终报告
用户
```

### 关键机制

#### 1. 主智能体的调度能力

主智能体不是简单的"传话筒"，它会：
- **理解任务全貌**：知道整个项目的目标
- **合理分解任务**：将大任务拆成可并行的小任务
- **智能调度执行**：根据依赖关系安排执行顺序
- **质量把控**：检查子智能体的输出，要求补充或修正

#### 2. 子智能体的并发执行

多个子智能体可以**同时工作**，而不是排队等待：
- 子智能体1搜索热点的同时
- 子智能体2可以并行查询基金数据
- 子智能体3可以并行进行技术分析

**效率提升**：原本串行需要30分钟的任务，并行只需要10分钟。

#### 3. 全流程透明化

在右侧的"工具信息"区域，你可以看到：
- **Plan**：主智能体的整体规划
- **SubAgents**：每个子智能体的执行状态
  - 哪个子智能体正在运行
  - 已经输出了什么结果
  - 调用了哪些工具
- **Tools**：具体的工具调用详情

![子智能体监控](/docs-images/subagent-monitor.png)

*说明：右侧显示子智能体列表，标注运行状态和输出结果*

**好处**：如果某个环节出问题，你可以精确定位；如果输出不符合预期，你可以针对性调整。

#### 4. 子智能体的独立性

每个子智能体都是独立的：
- 有独立的模型配置
- 有独立的系统提示词
- 可以绑定独立的工具库和知识库
- 有独立的上下文（不会互相干扰）

这让每个子智能体都能专注于自己的任务，发挥最佳性能。

## 4.4 自定义你的专家团队

自动生成的专家团队已经很好用，但你也可以根据需要进行调整。

### 如何修改专家配置？

萌芽的所有配置完全透明，你可以精细调整：

1. 进入"专家库"
2. 找到你想修改的智能体
3. 点击编辑按钮

![专家编辑](/docs-images/agent-edit.png)

*说明：展示专家库编辑界面，可修改各项参数*

### 可调整的参数

| 参数 | 调整建议 |
|---|---|
| **模型选型** | 推理任务用Kimi/Deepseek，创意任务用豆包，平衡速度和质量 |
| **系统提示词** | 细化角色定义，添加专业术语、工作流程、输出格式要求 |
| **工具库** | 根据任务需要增删工具，比如增加地图工具、减少金融工具 |
| **知识库** | 绑定相关的专业资料库，让专家能参考内部文档 |
| **子智能体** | 调整团队结构，增加或移除子智能体 |

### 保存和复用

专家团队**永久保存到专家库**：

你可以：
- 随时调用这个优化后的团队（在输入区域的专家库选择器中选择）
- 持续迭代优化，团队越用越顺手

这就像是从"每次都要重新招人"升级为"拥有一支常备军"。

## 4.5 萌芽专家模式 vs Openclaw多智能体

你可能想问：Openclaw也支持多智能体，和萌芽有什么区别？

| 对比维度 | Openclaw | 萌芽 | 关键差异 |
|---|---|---|---|
| **需求确认** | 主智能体"自己决定"创建什么子智能体，基于自己的理解直接执行，**不会反复和你确认**细节 | 通过**交互式规划**，反复和你确认需求细节，确保完全理解意图后再创建团队 | 萌芽从根源避免"答非所问"，Openclaw可能方向一开始就错 |
| **配置透明度** | **黑盒配置**，你看不到具体配置（模型、提示词、工具），也**无法手动调整** | **白盒配置**，所有配置完全透明，可在专家库中手动调整模型、系统提示词、工具、知识库等所有参数 | 萌芽完全由你掌控，Openclaw无法修改 |
| **保存与复用** | 专家是**临时的**，任务结束就"消失"，**无法永久保存**，也不能在其他对话中复用 | 专家团队**自动保存到专家库**，可在任何对话中随时调用，持续迭代优化 | 萌芽是"常备军"可沉淀复用，Openclaw是"临时搭班子"用完即散 |

**总结**：
- Openclaw是"AI觉得你需要什么，就给你什么"——**你可能不满意，但无法修改**
- 萌芽是"AI先问你想要什么，确认后再做，做完还可以改"——**完全由你掌控**

### 案例1：行业研究报告生成

|---|---|---|

| **报告撰写员** | 整合内容，生成规范报告 | 文档工具 |

**输出**：一份包含市场概况、竞争格局、政策环境、技术趋势、投资建议的完整报告。
